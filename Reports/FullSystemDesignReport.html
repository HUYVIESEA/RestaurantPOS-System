Báo cáo Thiết kế Hệ thống RestaurantPOS

Ngày tạo: 2025-12-11

Dự án: RestaurantPOS System

1. Phân rã Chức năng (Functional Decomposition)

1.1. Mức 1: Phân hệ & Mô-đun (Subsystems)

Hệ thống được chia thành 2 thành phần chính:

  • Desktop App: Ứng dụng bán hàng tại quầy.
  • Backend API: Hệ thống xử lý trung tâm.

(Sơ đồ cấu trúc tổng quan)

graph TD
    System[RestaurantPOS] --> Desktop[Desktop App]
    System --> API[Backend API]

    Desktop --> D_POS[Module Bán Hàng]
    Desktop --> D_Catalog[Module Thực Đơn]
    Desktop --> D_Report[Module Báo Cáo]
    Desktop --> D_System[Module Quản trị]

    API --> A_Auth[Service Xác thực]
    API --> A_Business[Service Nghiệp vụ]
    API --> A_Infra[Service Hạ tầng]

1.2. Mức 2: Chi tiết Chức năng (Detailed Functions)

Chi tiết các chức năng bên trong từng mô-đun:

A. Desktop Application

Module Chức năng Chính Chi tiết
Bán Hàng (POS) Quản lý Bàn Theo dõi trạng thái, Chuyển/Gộp bàn
Order Chọn món, Ghi chú, Topping, Takeaway
Thanh toán Tính tiền, VietQR động, In hóa đơn
Thiết bị In bếp, Màn hình khách hàng
Thực Đơn Quản lý Catalog CRUD Danh mục, Sản phẩm, Giá, Ảnh
Báo Cáo Dashboard Biểu đồ doanh thu, Món bán chạy

B. Backend API

  • Auth Service: Đăng nhập, Phân quyền (Role-based).
  • Business Service: Xử lý Orders, Tables, Products.
  • Infrastructure: Upload ảnh, VietQR Gen, SignalR (Real-time).

1.3. Mức 3: Quy trình Xử lý (Process Flows)

Mô tả luồng đi của các nghiệp vụ quan trọng.

Quy trình Đặt món (Order Flow)

1. Chọn Bàn -> Kiểm tra trạng thái
2. Nếu Bàn trống -> Mở Menu -> Chọn Món -> Note/Topping
3. Gửi Order -> API Validate -> Lưu DB
4. Cập nhật trạng thái Bàn -> SignalR báo Bếp -> In phiếu

Quy trình Thanh toán (Payment Flow)

1. Chọn Đơn -> Tính Tổng tiền
2. Chọn Phương thức:
   - Tiền mặt: Nhập tiền khách đưa -> Tính tiền thừa
   - VietQR: API tạo QR -> Khách quét -> Webhook xác nhận
3. Xác nhận Thanh toán -> API Update -> In Hóa đơn -> Giải phóng bàn

2. Thiết kế Cơ sở Dữ liệu (Database Schema)

Các thực thể chính trong hệ thống:

2.1. Quản lý Người dùng

  • Users: ID, Username, PasswordHash, FullName, Role (Admin/Staff/Manager).
  • UserDevices: Quản lý token thiết bị (Push Notification).

2.2. Vận hành (Operations)

  • Tables: Số bàn, Sức chứa, Trạng thái, Khu vực (Tầng).
  • Orders: Mã đơn, Tổng tiền, Trạng thái (Pending/Completed), Loại đơn.
  • OrderItems: Chi tiết món, Số lượng, Giá, Ghi chú.

2.3. Danh mục & Tài chính

  • Products: Tên, Giá, Ảnh, Danh mục ID.
  • Payments: Lịch sử thanh toán, Phương thức, Transaction ID.
  • PaymentSettings: Cấu hình tài khoản ngân hàng nhận tiền.

3. Biểu đồ Use Case

Tác nhân (Actors):

  1. Nhân viên (Staff): Thao tác bán hàng, order, thanh toán.
  2. Quản lý (Manager): Quản trị menu, nhân sự, xem báo cáo (Kế thừa quyền Staff).

(Danh sách Use Case chính)

  • Hệ thống: Đăng nhập, Đổi mật khẩu.
  • POS: Quản lý bàn, Gọi món, Thanh toán, In hóa đơn.
  • Admin: Quản lý thực đơn, Cấu hình thanh toán, Xem Dashboard.
