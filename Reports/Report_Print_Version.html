CHƯƠNG 1: TỔNG QUAN ĐỀ TÀI

1.1. Lý do chọn đề tài

Trong lĩnh vực kinh doanh nhà hàng, tốc độ và sự chính xác là hai yếu tố then chốt. Các hệ thống POS (Point of Sale) truyền thống thường hoạt động offline, khó khăn trong việc đồng bộ dữ liệu hoặc mở rộng nhiều chi nhánh. Ngược lại, các hệ thống thuần web lại phụ thuộc quá nhiều vào trình duyệt và đôi khi không tận dụng được hết sức mạnh phần cứng để xử lý các tác vụ nặng như in ấn tức thời hay hiển thị đồ họa phức tạp mượt mà.

Đề tài "Xây dựng hệ thống quản lý nhà hàng (Desktop POS & API)" hướng tới việc kết hợp sức mạnh của ứng dụng Desktop (WPF) với tính linh hoạt của kiến trúc Microservices/API. Hệ thống đảm bảo nhân viên thu ngân có một công cụ làm việc hiệu năng cao, giao diện thân thiện, đồng thời dữ liệu được quản lý tập trung, bảo mật trên Server.

1.2. Mục tiêu của đề tài

  • Về mặt kiến thức: Làm chủ .NET 8, C#, Architecture RESTful API, WPF, MVVM Pattern, SignalR, JWT.
  • Về mặt ứng dụng: Xây dựng phần mềm quản lý toàn diện quy trình: Đặt bàn -> Gọi món -> Chế biến -> Thanh toán.

1.3. Phạm vi nghiên cứu

  1. RestaurantPOS.API (Backend): Cung cấp API, Auth, Data (PostgreSQL).
  2. RestaurantPOS.Desktop (Client): App Windows cho Thu ngân/Quản lý.

1.4. Đối tượng sử dụng

  • Admin: Cấu hình hệ thống, quản lý.
  • Thu ngân: Bán hàng, thanh toán.
  • Bếp: Xem đơn món, trả món.

CHƯƠNG 2: CƠ SỞ LÝ THUYẾT VÀ CÔNG NGHỆ

2.1. Nền tảng Microsoft .NET Core / .NET 6+

.NET là nền tảng phát triển mã nguồn mở, đa nền tảng, hiệu năng cao.

2.2. Kiến trúc RESTful Web API

Stateless, Resource-based, chuẩn HTTP Methods (GET, POST, PUT, DELETE).

2.3. Công nghệ WPF & MVVM

WPF sử dụng XAML để thiết kế giao diện vector sắc nét. MVVM giúp tách biệt logic (ViewModel) và giao diện (View), tăng khả năng test và bảo trì.

2.4. Entity Framework Core (Code First)

ORM mạnh mẽ, sử dụng cú pháp LINQ để truy vấn DB. Code First Migrations giúp quản lý schema DB từ code C#.

2.5. Real-time với SignalR

Công nghệ đẩy dữ liệu thời gian thực (WebSockets). Dùng để thông báo trạng thái bàn ăn ngay lập tức cho các máy khác.

5. OrderItems: OrderId, ProductId, Quantity.

CHƯƠNG 4: TRIỂN KHAI XÂY DỰNG ỨNG DỤNG

4.1. Xây dựng Backend (API)

Cấu hình Program.cs, DI Container, Database Connection, JWT Authentication.

Xây dựng OrderService xử lý logic đặt món và tính toán giá tiền.

4.2. Xây dựng Client (Desktop)

Tổ chức theo cấu trúc MVVM: Views, ViewModels, Models, Services.

Sử dụng MaterialDesign để làm đẹp giao diện.

Tích hợp SignalR Client để nhận thông báo real-time từ server.

CHƯƠNG 5: KẾT QUẢ VÀ KẾT LUẬN

5.1. Kết quả đạt được

Hệ thống hoạt động ổn định, đáp ứng nghiệp vụ cơ bản: Đặt bàn, Gọi món, Thanh toán QR Code, Báo cáo.

5.2. Hướng phát triển

- Hỗ trợ Offline với SQLite.

- Phát triển ứng dụng Mobile cho nhân viên phục vụ.

- Tích hợp phần cứng: Máy in nhiệt, Cash drawer.
